# План курса: Уроки 31-47

| № | Название урока | Основные темы |
|---|---------------|---------------|
| **31** | **SQL** | CTE, индексы (CREATE INDEX), представления (Views, Materialized View), ограничения целостности (FOREIGN KEY, CHECK, UNIQUE) |
| **32** | **Статистика для аналитика** | Распределения, центральные тенденции, проверка гипотез (p-value, t-тест, chi-squared), корреляция и причинность, типичные ошибки интерпретации |
| **33** | **Комплексный анализ данных** | Полный цикл EDA на реальном датасете: очистка, агрегация, визуализация, выводы, feature engineering как подготовка данных |
| **34** | **ML: лучшие практики** | Cross-validation (зачем и как), overfitting/underfitting, регуляризация (L1/L2 концептуально), несбалансированные данные, Pipeline |
| **35** | **Streamlit: основы** | Установка, виджеты (slider, selectbox, file_uploader), отображение данных, визуализация, layout (sidebar, columns) |
| **36** | **Streamlit: ML приложения** | Интеграция ML моделей, кэширование (@st.cache_data), многостраничные приложения, deployment на Streamlit Cloud |
| **37** | **A/B тестирование** | Что такое A/B тест, формулировка гипотезы, выбор метрики, расчёт размера выборки, интерпретация результатов, типичные ошибки в бизнесе |
| **38** | **Временные ряды** | Тренд, сезонность, шум, декомпозиция (STL), rolling mean/std, прогнозирование, примеры: продажи, трафик, финансы |
| **39** | **Источники данных и автоматизация** | Работа с API (requests), автоматизация отчётов (pandas + openpyxl), откуда берут данные в реальных проектах |
| **40** | **Data Storytelling** | Как презентовать результаты анализа, структура отчёта/презентации, выбор правильного типа визуализации, коммуникация с нетехнической аудиторией |
| **41** | **Нейросети: как это работает** | Концепция нейросети, слои, функции активации, обучение, примеры применения |
| **42** | **Консультация** | Консультация по проекту, возможные направления в сфере Data |
| **43-47** | **Итоговый проект** | Выбор темы, источники данных, формулировка задачи, критерии оценки, требования к проекту, выполнение проекта, индивидуальные консультации |
