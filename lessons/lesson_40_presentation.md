# REST API: интуитивная модель

Представьте ресторан:

- клиент - это ваш Python-скрипт
- официант - это API
- кухня - это внутренняя база данных
- меню - это документация API

Клиент не лезет на кухню напрямую. Он делает заказ через стандартизированный интерфейс.

---

# Что означает REST

REST (Representational State Transfer) - это набор архитектурных принципов для веб-сервисов.

Ключевые идеи:
- клиент и сервер разделены
- каждый запрос содержит все нужные данные
- сервер не обязан помнить состояние прошлого запроса
- данные обычно возвращаются в JSON

---

# HTTP-методы

Для аналитика чаще всего нужен `GET`.

| Метод | Что делает | Типичный сценарий |
|------|------------|-------------------|
| `GET` | Получает данные | Загрузить прогноз, курс валют, статистику |
| `POST` | Создаёт данные | Отправить форму, создать запись |
| `PUT` | Обновляет данные | Изменить существующую запись |
| `DELETE` | Удаляет данные | Удалить объект |

---

# Почему JSON удобен

JSON хорошо ложится на Python:

- объект JSON похож на `dict`
- массив JSON похож на `list`
- вложенные структуры легко распаковать в `DataFrame`

Для сложных ответов удобно использовать `pd.json_normalize()`.

---

# Что обязательно проверять в API

Перед обработкой данных нужно смотреть не только на тело ответа, но и на статус-код.

| Код | Значение | Что делать |
|-----|----------|------------|
| 200 | Успех | продолжать обработку |
| 400 | Ошибка запроса | проверить параметры |
| 401/403 | Нет доступа | проверить ключи и права |
| 404 | Ресурс не найден | проверить URL и endpoint |
| 429 | Слишком много запросов | добавить паузу и retry |
| 500 | Ошибка сервера | повторить позже, записать в лог |

---

### Пример запроса через requests

```python
import requests

url = "https://api.open-meteo.com/v1/forecast"
params = {
    "latitude": 35.6895,
    "longitude": 139.6917,
    "hourly": "temperature_2m",
    "forecast_days": 3,
    "timezone": "auto",
}

response = requests.get(url, params=params, timeout=10)
response.raise_for_status()
data = response.json()
```

---

# Почему важен timeout

Без `timeout` скрипт может зависнуть на неопределенное время.

Это особенно опасно в автоматизации:
- один медленный запрос тормозит весь pipeline
- пользователь думает, что программа сломалась
- в логике оркестрации появляются зависшие задачи

---

# Как читать ответ API

Обычно ответ имеет несколько уровней вложенности.

Пример для погодного API:

- верхний уровень - метаданные
- вложенный блок `hourly` - массивы времени и значений
- каждое значение должно быть сопоставимо по длине

Если ключей нет, это не DataFrame, а ошибка структуры ответа.

---

# Валидируем данные до анализа

Перед тем как строить графики или отчеты, проверяем:

1. факт успешной загрузки
2. `shape` результата
3. наличие ключевых колонок
4. пропуски и аномалии

Это же правило работает и для файлов, и для API-ответов.

# Преобразование JSON в DataFrame

```python
import pandas as pd

hourly = data.get("hourly", {})

df = pd.DataFrame({
    "timestamp": hourly.get("time", []),
    "temperature_celsius": hourly.get("temperature_2m", []),
})

df["timestamp"] = pd.to_datetime(df["timestamp"])
```

Смысл простой: из вложенного JSON делаем плоскую таблицу для анализа.

# Как выбрать API для урока

| Критерий | Что ищем |
|----------|----------|
| Доступность | работает без сложной авторизации |
| Структура | есть вложенные поля и массивы |
| Аналитическая ценность | можно строить графики и сравнения |
| Ограничения | понятные лимиты и документация |

Для аналитики особенно полезны источники с временными рядами.

## Автоматизация отчетов: зачем она нужна

Обычно бизнес не читает сырой код и не работает в Jupyter.

Нужен артефакт, который:
- легко открыть в Excel
- уже отформатирован
- готов к пересылке и обсуждению

Автоматизация убирает ручное форматирование и снижает количество ошибок.

---

# CSV и Excel

CSV подходит для обмена данными, но плохо подходит для презентации результата.

Проблемы CSV:
- нет листов
- нет форматирования
- типы данных могут интерпретироваться по-разному
- сложнее делать корпоративный отчет

---

# Когда использовать openpyxl и XlsxWriter

| Библиотека | Сильная сторона | Когда брать |
|------------|----------------|-------------|
| `openpyxl` | чтение и изменение существующих `.xlsx` | если надо править готовый файл |
| `XlsxWriter` | создание красивых новых файлов | если отчет генерируется с нуля |

Для учебного шаблона важно понимать обе возможности.

---

# Пример генерации отчета

```python
import pandas as pd

with pd.ExcelWriter("report.xlsx", engine="xlsxwriter") as writer:
    df.to_excel(writer, sheet_name="Summary", index=False)

    workbook = writer.book
    worksheet = writer.sheets["Summary"]

    header_format = workbook.add_format({
        "bold": True,
        "fg_color": "#003366",
        "font_color": "#FFFFFF",
        "border": 1,
    })
```

---

# Что делает отчет профессиональным

- понятные заголовки
- автоширина колонок
- закрепление верхней строки
- автофильтр
- единый визуальный стиль

Пользователь должен открыть файл и сразу начать работать с данными.
