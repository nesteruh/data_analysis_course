# Ensemble Learning
## Ансамбли моделей машинного обучения

---
# Что такое Ensemble Learning?

Несколько моделей работают лучше, чем одна

**Ансамбль (Ensemble)** - комбинация нескольких моделей для улучшения:
- Точности предсказаний
- Стабильности результатов
- Устойчивости к переобучению

**Принцип:** объединяем слабые модели -> получаем сильную модель

---
### Основные типы ансамблей

**1. Bagging (Bootstrap Aggregating)**
- Параллельное обучение моделей
- Снижает variance
- Пример: **Random Forest**

**2. Boosting**
- Последовательное обучение моделей
- Снижает bias
- Примеры: AdaBoost, Gradient Boosting, XGBoost


---
# Bagging: Bootstrap Aggregating

**Алгоритм:**
1. Создаем бутстрап-выборки (случайные выборки с возвращением)
2. Обучаем модель на каждой выборке
3. Усредняем/голосуем предсказания

- Уменьшает переобучение
- Работает параллельно
- Устойчив к выбросам

---
# Bagging: схема работы

```
Обучающая выборка (N объектов)
         ↓
Бутстрап-выборка 1 -> Модель 1 -> Предсказание 1
Бутстрап-выборка 2 -> Модель 2 -> Предсказание 2    -> Агрегация
Бутстрап-выборка 3 -> Модель 3 -> Предсказание 3       (голосование/
         ...                         ...                усреднение)
Бутстрап-выборка M -> Модель M -> Предсказание M
```

**Random Forest** - это Bagging с деревьями решений + случайный выбор признаков

---
# Boosting: общая идея

**Принцип:** последовательное обучение, где каждая новая модель исправляет ошибки предыдущих

**Ключевые отличия от Bagging:**
- Модели обучаются **последовательно**, а не параллельно
- Больший вес на ошибочно классифицированные объекты
- Финальное предсказание = взвешенная сумма

**Цель:** превратить слабые модели (weak learners) в сильную модель

---
# AdaBoost (Adaptive Boosting)

**Алгоритм:**
1. Начинаем с равных весов всех объектов
2. Обучаем базовую модель
3. Увеличиваем веса неправильно классифицированных объектов
4. Обучаем следующую модель с новыми весами
5. Повторяем N раз


---
# AdaBoost: схема работы

```
Итерация 1: Равные веса -> Модель 1 -> Ошибки
                ↓
Итерация 2: Больше вес на ошибки -> Модель 2 -> Ошибки
                ↓
Итерация 3: Больше вес на ошибки -> Модель 3 -> Ошибки
                ↓
              ...
                ↓
Финальная модель = взвешенная комбинация всех моделей
```

**Особенность:** фокусируется на "трудных" примерах

---
# Gradient Boosting

**Идея:** каждая новая модель аппроксимирует отрицательный градиент функции потерь по текущим предсказаниям.

1. Обучаем базовую модель $F_0(x)$
2. Вычисляем остатки: $r_1 = y - F_0(x)$
3. Обучаем модель на остатках: $h_1(x) \approx r_1$
4. Обновляем: $F_1(x) = F_0(x) + \eta \cdot h_1(x)$
5. Повторяем для $F_2, F_3, ..., F_N$

---
# Gradient Boosting: визуализация

```
y_true = 100

F_0(x) = 70  ->  Ошибка (residual) = 30
     ↓
h_1(x) = 25  ->  F_1(x) = 70 + 0.1*25 = 72.5
     ↓                ↓
                  Ошибка = 27.5
     ↓
h_2(x) = 22  ->  F_2(x) = 72.5 + 0.1*22 = 74.7
     ↓                ↓
                  Ошибка = 25.3
  ```

С каждой итерацией приближаемся к истинному значению

---
# XGBoost (Extreme Gradient Boosting)


**Улучшения по сравнению с классическим GBM:**
- Регуляризация (L1, L2) для предотвращения переобучения
- Параллельные вычисления
- Работа с пропущенными значениями
- Встроенная кросс-валидация
- Early stopping

---
# XGBoost: ключевые параметры

- `n_estimators` - количество boosting раундов

- `max_depth` - максимальная глубина дерева
- `min_child_weight` - минимальная сумма весов в листе

- `learning_rate` (η) - шаг обучения (обычно 0.01-0.3)
- `subsample` - доля выборки для каждого дерева

- `reg_alpha` - L1 регуляризация
- `reg_lambda` - L2 регуляризация

---
**Рекомендации:**
- `learning_rate = 0.1`, `n_estimators = 100` - для начала
- `learning_rate = 0.01`, `n_estimators = 1000` - для лучшего качества
- Используйте `early_stopping` для автоматического выбора

```python
model = XGBClassifier(n_estimators=1000, learning_rate=0.01)
model.fit(X_train, y_train, 
          eval_set=[(X_val, y_val)],
          early_stopping_rounds=50)
```

---
# Сравнение Bagging vs Boosting

| Характеристика | Bagging | Boosting |
|---------------|---------|----------|
| **Обучение** | Параллельное | Последовательное |
| **Что снижает** | Variance | Bias |
| **Скорость** | Быстрее | Медленнее |
| **Переобучение** | Меньше риск | Может переобучиться |
| **Примеры** | Random Forest | AdaBoost, GBM, XGBoost |
| **Веса объектов** | Равные | Адаптивные |

# Feature Importance в ансамблях

**В деревьях и ансамблях:**
- Измеряется уменьшением impurity (Gini/Entropy)
- Чем больше признак используется -> тем важнее

**Применение:**
- Feature selection
- Понимание данных
- Интерпретация модели

---


**1. Когда использовать Bagging:**
- Модель склонна к переобучению (high variance)
- Данные зашумлены
- Нужна параллельная обработка

**2. Когда использовать Boosting:**
- Модель недообучается (high bias)
- Нужна максимальная точность
- Есть время на последовательное обучение
