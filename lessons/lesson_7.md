# Масштабирование данных

---
# Масштабирование признаков
Масштабирование признаков - это метод, используемый для нормализации диапазона признаков данных. В обработке данных он также известен как нормализация данных и обычно выполняется на этапе предварительной обработки данных

---

# Основная идея
Сырые числовые признаки похожи на “взрослых в мире, построенном для детей”:
- одни огромные (500 000)
- другие крошечные (0.01)

модель “не понимает”, кто важнее.
Scaling = приведение признаков “к одинаковому масштабу”.
Цель - модель должна видеть информацию, а не размер числа.

---
# Где применять scaling

Где scaling не нужен:
- категориальные признаки
- бинарные признаки (0/1)
- счётчики
- циклические признаки

---

# Scaling нужен для:

- числовых признаков с разными диапазонами
- признаков в разных единицах
- признаков с очень большим разбросом
- skewed features

---

# Алгоритмы 
Создание признаков для алгоритмов:
- kNN
- SVM
- Logistic Regression
- Linear Regression
- Нейронные сети

---
# Зачем нужен scaling?
- Чтобы признаки были равны по влиянию
    - Большой признак может “затмить” важный маленький.
- Улучшение качества моделей
- Избежание численной нестабильности
    - Модели могут ломаться от огромных значений.
- Интерпретация коэффициентов в регрессии
    - Коэффициенты становятся сопоставимыми.

---

1) Min-Max Scaling
    - Приводит значения к диапазону [0;1].
2) Standard Scaling
    - Центрирование к mean=0, std=1.
3) Robust Scaling
    - Использует медиану и IQR, устойчив к выбросам.
4) Log Transformation
5) Box-Cox Transformation
    - Автоматически “нормализует” распределение под нормальное.

---
| Метод    | Устойчив к выбросам | Делает нормальным | Фиксирует диапазон | Когда применять                 |
| -------- | ------------------- | ----------------- | ------------------ | ------------------------------- |
| MinMax   | ➖                   | ➖                 | ➕                  | перед NN, если нет выбросов     |
| Standard | ➖                   | иногда            | ➖                  | линейные модели, SVM            |
| Robust   | ➕                   | ➖                 | ➖                  | есть выбросы                    |
| Log      | ➕                   | частично          | ➖                  | правый хвост                    |
| Box-Cox  | ➕                   | ➕                 | ➖                  | нужно приближение к нормальному |
