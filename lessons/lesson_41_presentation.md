# Траектории развития и архитектура итоговых проектов в Data


---

# Переход мышления

Было:
- "Как запустить алгоритм?"

Должно стать:
- "Какую измеримую ценность это решение даст бизнесу?"

`бизнес-вопрос -> гипотеза -> данные -> метод -> решение -> действие`

---

# Карьерные треки 

1. Data Analyst
2. BI Analyst
3. Product Analyst
4. Data Scientist

У каждого трека разный фокус, стек и формат итогового проекта.

---

# 1. Junior Data Analyst

**Что делает:**
- Анализирует прошлые и текущие данные.
- Ищет причины изменений метрик.
- Делает отчеты и рекомендации для бизнеса.

**Типовые задачи:**
- EDA, очистка, ad-hoc анализ, поиск аномалий.

---

# Стек Data Analyst

- SQL 
- Python: `pandas`, `numpy`, `matplotlib`, `seaborn`
- Excel / Google Sheets
- BI-визуализация: Power BI / Tableau / Looker Studio
- Статистика

**Что показать в портфолио:**
- Аналитический проект с чисткой данных, EDA и бизнес-выводами.

---

# 2. Junior BI Analyst

**Что делает:**
- Проектирует отчетность и KPI-дашборды.
- Строит модели данных для мониторинга бизнеса.
- Автоматизирует управленческую аналитику.

**Типовые задачи:**
- Витрины данных, дашборды, мониторинг метрик.

---

# Стек BI Analyst

- BI-платформы: Power BI / Tableau / Looker
- SQL 
- DAX, Power Query (M)
- Моделирование данных: Star Schema
- Базовое понимание DWH

**Что показать в портфолио:**
- Корпоративный дашборд с продуманной моделью данных.

---

# 3. Junior Product Analyst

**Что делает:**
- Анализирует поведение пользователей в продукте.
- Оценивает влияние фич на метрики.
- Помогает принимать продуктовые решения.

**Типовые задачи:**
- Воронки, когорты, retention, A/B-тесты.

---

# Стек Product Analyst

- SQL по сырым логам событий
- Python и статистика для проверки гипотез
- BI-инструменты для воронок и retention
- Продуктовые платформы: Amplitude / Mixpanel / GA4
- Экспериментальные платформы A/B (по компании)

**Что показать в портфолио:**
- Полный разбор A/B-теста с бизнес-решением.

---

# 4. Junior Data Scientist

**Что делает:**
- Строит предиктивные модели.
- Автоматизирует решения через ML.
- Разворачивает модели для использования в продукте.

**Типовые задачи:**
- Классификация, регрессия, прогноз, сегментация, скоринг.

---

# Стек Data Scientist

- Python + ML: `scikit-learn`, `xgboost`
- DL-фреймворки: PyTorch / TensorFlow (по задаче)
- SQL для подготовки выборок
- Развертывание: Streamlit, API-сервисы
- Облака и Big Data: AWS/GCP/Azure, Spark/Databricks (зона роста)
- Современный AI-слой: LLM API, LangChain (опционально)

**Что показать в портфолио:**
- End-to-end ML-проект с интерфейсом или API.

---

# Modern Data Stack (обзорно)

- Хранилища: Snowflake, BigQuery, ClickHouse
- Оркестрация: Airflow
- Интеграция и трансформации: ETL/ELT-инструменты
- AI-ассистенты для SQL/Python

# Как выбрать тему проекта правильно

Не так:
- "Хочу сделать Random Forest"

Так:
1. Какую бизнес-проблему решаем?
2. Какую гипотезу проверяем?
3. Какие данные доступны?
4. Какая метрика успеха?
5. Как бизнес будет использовать результат?

---

# Что такое хороший итоговый проект

Проект считается сильным, если в нем есть:

1. реальный вопрос без очевидного ответа;
2. реалистичные данные (API/логи/выгрузки/скрейпинг);
3. валидация данных до анализа;
4. корректный метод и проверка качества;
5. понятный артефакт для бизнеса (дашборд, приложение, отчет);
6. четкая интерпретация ограничений и рисков.

---

# Мини-архитектура финального проекта

1. Business Problem
2. Data Collection (CSV/API/SQL/Scraping)
3. Data Validation + EDA
4. Modeling / Statistical Inference
5. Evaluation (метрики + ограничения)
6. Delivery Layer (Power BI / Streamlit / отчет)
7. Business Recommendation
