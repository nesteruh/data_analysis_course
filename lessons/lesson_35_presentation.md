# Streamlit: основы



---

# Почему это важно аналитикам

До Streamlit:
- анализ существует только в ноутбуке
- результат сложно понять не аналитику

После Streamlit:
- тот же Python-код становится приложением
- менеджер или клиент сам выбирает фильтры
- аналитика становится продуктом, а не только отчётом

---

# Что такое Streamlit?

**Streamlit** - это Python-фреймворк для быстрого создания веб-приложений для анализа данных и ML.

**Почему это удобно для аналитика:**
- не нужен фронтенд (HTML/CSS/JS)
- код на чистом Python
- быстрый прототип за 15 – 30 минут
- легко показать результат заказчику

---

# Интуитивная модель Streamlit

Главная идея:

1. Вы пишете Python-скрипт сверху вниз
2. Streamlit рисует интерфейс по этому коду
3. Пользователь меняет виджет
4. Скрипт **перезапускается** и пересчитывает экран

>  UI всегда отражает текущее состояние кода и данных.


---

# Где Streamlit полезен

- интерактивные дашборды
- внутренние аналитические инструменты
- demo ML-моделей
- прототип MVP перед полноценной разработкой


---

# Где Streamlit не лучший выбор

- сложный публичный продукт с десятками экранов и сложной авторизацией
- heavy front-end анимации и нестандартный UX

Для учебных и аналитических задач - почти всегда отличный выбор.

# Базовый каркас приложения

```python
import streamlit as st

st.set_page_config(page_title="EDA App", layout="wide")
st.title("Мини EDA приложение")
st.write("Первое приложение на Streamlit")
```

---

# Как читать этот код интуитивно

- `set_page_config` - базовые настройки страницы
- `title` - главный акцент страницы
- `write` - универсальный вывод (текст, таблица, объекты)

Порядок строк в коде = порядок блоков на экране.

---

# Главные компоненты интерфейса

| Компонент | Пример | Зачем нужен |
|----------|--------|-------------|
| `st.title` | Заголовок | Структура страницы |
| `st.write` | Текст / объект | Универсальный вывод |
| `st.dataframe` | Таблица | Просмотр данных |
| `st.metric` | KPI карточка | Быстрые метрики |
| `st.bar_chart` | График | Базовая визуализация |

---

# Виджеты = входные параметры приложения

Интуитивно виджет можно воспринимать как переменную:

```python
age = st.slider("Возраст", 18, 65, 30)
```

`age` - это текущее значение, которое выбрал пользователь.

---

# Виджеты ввода

```python
city = st.selectbox("Выберите город", ["Almaty", "Astana", "Shymkent"])
age = st.slider("Возраст", min_value=18, max_value=65, value=30)
show_raw = st.checkbox("Показать исходные данные")
```

**Идея:** любое изменение виджета автоматически перезапускает скрипт и обновляет интерфейс.


---

# Sidebar и layout

```python
with st.sidebar:
    st.header("Фильтры")
    min_age = st.slider("Минимальный возраст", 18, 65, 25)

col1, col2 = st.columns(2)
with col1:
    st.metric("Строк", 1250)
with col2:
    st.metric("Колонок", 18)
```

---

# Практический принцип layout

- `sidebar` - фильтры и настройки
- основная область - графики, таблицы, KPI

Так пользователь быстро понимает логику:
«слева управление -> справа результаты».

# Загрузка данных пользователем

```python
uploaded_file = st.file_uploader("Загрузите CSV", type=["csv"])

if uploaded_file is not None:
    df = pd.read_csv(uploaded_file)
    st.success("Файл загружен")
    st.dataframe(df.head())
```

---

# Поток данных в приложении

1. Пользователь загружает CSV
2. Вы читаете его в `DataFrame`
3. Проверяете качество и структуру
4. Применяете фильтры
5. Показываете метрики и графики


---

# Валидация датасета (обязательно)

Перед анализом всегда проверяем:
1. путь / факт загрузки файла
2. размерность (`shape`)
3. пропуски (`isnull().sum()`)
4. обязательные колонки (в т.ч. target, если нужен)

```python
required_cols = ["Age", "Salary"]
missing_cols = [c for c in required_cols if c not in df.columns]
```

---

# Что делать, если валидация не прошла

- показать понятную ошибку пользователю
- остановить расчёты до исправления данных
- не строить графики на некорректных колонках

Пример сообщения:
`Отсутствуют обязательные колонки: ['Age']`

---

# Мини-шаблон безопасной проверки

```python
if uploaded_file is None:
    st.info("Загрузите CSV")
    st.stop()

if missing_cols:
    st.error(f"Нет обязательных колонок: {missing_cols}")
    st.stop()
```

`st.stop()` — простой способ прервать выполнение, если входные данные некорректны.

---

# `st.cache_data` для ускорения

```python
@st.cache_data
def load_data(file):
    return pd.read_csv(file)
```

**Зачем:**
- не читать файл заново на каждый интерактивный шаг
- ускорить приложение

---

# Интуиция про кеш

Без кеша:
- на каждом клике файл читается заново
- интерфейс начинает тормозить

С кешем:
- Streamlit повторно использует уже рассчитанный результат
- приложение работает заметно быстрее

---

# Когда кеш особенно нужен

- чтение больших CSV/Excel
- тяжёлые группировки и агрегации
- частое переключение фильтров

---

# Пошаговая сборка приложения 

Шаг 1: каркас и заголовки  
Шаг 2: загрузка CSV  
Шаг 3: валидация (shape, missing, columns)  
Шаг 4: фильтры в sidebar  
Шаг 5: таблица + метрики  
Шаг 6: график  

---

# Практика на уроке

Собираем приложение с блоками:
1. Upload CSV
2. Валидация структуры данных
3. Просмотр `head()`, `describe()`, пропусков
4. Фильтр по категориальному признаку
5. Гистограмма выбранной числовой колонки

# Пример графика

```python
num_col = st.selectbox("Числовая колонка", df.select_dtypes(include="number").columns)
hist = pd.cut(df[num_col].dropna(), bins=30).value_counts().sort_index()
hist_df = hist.rename_axis("bin").reset_index(name="count")
hist_df["bin"] = hist_df["bin"].astype(str)
st.bar_chart(hist_df.set_index("bin")["count"])
```
