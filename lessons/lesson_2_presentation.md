# Git/Github
- Git - это **распределённая система контроля версий**
- GitHub - это веб-платформа для хостинга **Git-репозиториев**
---
**Система контроля версий** - это набор инструментов, отслеживающих историю набора файлов. В любой момент можно сказать системе контроля версий сохранить текущее состояние файлов.

Сохранение состояния похоже на создание резервной копии рабочей директории. В Git такое сохранение состояния называется **коммитом**.

---
**Репозиторий** - это место, где хранится проект и вся история изменений, управляемых системой контроля версий
# Контроль версий
- сохраняет историю изменений файлов;
- позволяет вернуться к любой предыдущей версии;
- помогает отслеживать, кто и когда изменил код. 

---
**Распределённый контроль версий** - это такая система контроля версий, в которой каждый разработчик хранит у себя полную копию **репозитория** со всей историей изменений

- у каждого есть полный бэкап (резервная копия) проекта;
- можно работать офлайн;
- обмен изменениями происходит при необходимости **push/pull**

---
# Глоссарий Git/GitHub (1)

| Термин | Определение |
|--------|-------------|
| **Репозиторий** | Хранилище проекта и всей истории изменений |
| **Локальный репозиторий** | Репозиторий на вашем компьютере |
| **Удалённый репозиторий** | Репозиторий на сервере |
| **.git** | Служебная папка с данными Git |
| **.gitignore** | Список файлов/папок, которые Git игнорирует |

---

# Глоссарий Git/GitHub (2)

| Термин | Определение |
|--------|-------------|
| **Commit (коммит)** | Сохранённое состояние проекта  |
| **Commit message** | Описание изменений |
| **SHA / Hash** | Уникальный идентификатор коммита |
| **History / Log** | История коммитов |
| **HEAD** | Указатель на текущую активную ветку/коммит |

---

# Глоссарий Git/GitHub (3)

| Термин | Определение |
|--------|-------------|
| **Working directory** | Файлы, с которыми вы работаете |
| **Staging area / Index** | Область подготовки файлов к коммиту |
| **Tracked / Untracked** | Отслеживаемые / неотслеживаемые файлы |
| **Stage (git add)** | Добавить в область подготовки |
| **Branch (ветка)** | Параллельная история разработки |
| **Main / Master** | Основная ветка проекта |
| **Merge (слияние)** | Объединение веток |

---

# Глоссарий Git/GitHub (4)

| Термин | Определение |
|--------|-------------|
| **Clone** | Скопировать удалённый репозиторий |
| **Pull** | Получить и применить удалённые изменения с сервера |
| **Fetch** | Получить изменения без применения |
| **Push** | Отправить свои изменения на GitHub|
| **Pull Request (PR)** | Запрос на принятие изменений |

# Основные команды Git
```bash
git init # Создание/реинициализация репозитория
git config --global user.name "Ваш username" # "логин"
git status # Просмотр состояния
git add <file> # Добавление файла в staging
git add . # Добавление всех файлов в staging
git commit -m "Сообщение коммита" # Создание коммита 
git log # Просмотр истории
```
---
# Взаимодействие с ветками
```bash
git branch <branch-name> # Создание новой ветки
git checkout <branch-name> # Переключение на ветку
git checkout -b <branch-name> # Создать и перейти
```

```bash
git merge <branch-name> # Слияние ветки
```
---

# Взаимодействие с репозиторием
``` bash
git remote add origin <url> # Подключение удалённого репозитория
git push # Отправка изменений на сервер
git push -u origin <branch-name> # "Первый пуш"
git pull # Получение изменений с сервера
git fetch # Получить изменений без слияния(применения)
git clone <url> # Клонирование удаленного репозитория
```
---
# Branch Workflow

 **Branch** — это отдельная линия разработки  

```bash
git checkout -b feature-cleaning
git add .
git commit -m "Добавил очистку данных"
git push origin feature-cleaning
```

---
# Хорошие практики при работе с Git
1) Качественные коммиты
 - Подробные сообщения, 1 логическое изменение
2) Всегда делайте 'pull' перед началом работы для избежания конфликтов
3) Чувствительные данные (CSV данные, пароли, токены) в .gitignore
